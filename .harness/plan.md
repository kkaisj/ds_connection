# Harness Plan

## Objective
在 4 个 Sprint 内完成数据连接器 MVP：连接市场、账号管理、任务闭环、存储与通知。

## Milestones
1. Sprint 1：基础骨架与核心实体 CRUD。
2. Sprint 2：任务调度、执行状态机、失败重试。
3. Sprint 3：多存储适配与通知中心。
4. Sprint 4：管理后台、观测看板、上线准备。

## Risks
1. 平台页面变化快导致采集逻辑失效。
2. 验证码/风控导致账号失效。
3. 高峰任务并发导致执行堆积。

## Mitigation
1. 适配器版本化 + 回归用例。
2. 账号健康评分 + 人工兜底。
3. 队列限流 + 优先级调度 + 指数退避重试。
