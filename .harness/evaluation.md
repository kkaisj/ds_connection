# Evaluation

## Scorecard
1. 功能完整性（30 分）：验收用例通过率。
2. 稳定性（25 分）：异常处理、重试策略、状态一致性。
3. 架构合规（20 分）：是否违反模块边界和依赖规则。
4. 可维护性（15 分）：测试质量、代码可读性、文档一致性。
5. 可观测性（10 分）：日志、指标、告警闭环。

## Release Gate
1. 总分 >= 80。
2. 任一维度不得低于 60。
3. 无 P0/P1 问题。

## Defect Priority
1. P0：数据丢失、越权访问、核心链路不可用。
2. P1：主要功能不可用或结果错误。
3. P2：功能可用但体验差或存在局部缺陷。
4. P3：轻微问题或文档问题。
